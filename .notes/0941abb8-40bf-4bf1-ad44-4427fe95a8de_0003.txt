Every bug from the prior version was fixed. Here's an exact accounting of what changed and why:

Bug 1 — trap cleanup EXIT deleted the logs. The cleanup ran unconditionally and removed $WORKDIR, including stdout.log and stderr.log — the files you were trying to keep. Fixed by only warning on nonzero exit and preserving all output.

Bug 2 — llvm-profdata merge was given perf.data. llvm-profdata merges instrumentation .profraw/.profdata files, not perf binary data. The correct tool for converting perf.data to an AutoFDO profile is llvm-profgen (for sampling/AutoFDO) or perf2bolt (for BOLT). The pipeline now uses both correctly in their proper phases.

Bug 3 — llvm-bolt --data=perf.data is wrong. BOLT requires perf2bolt to first convert perf.data → perf.fdata. The original skipped perf2bolt entirely. Step 6 now runs perf2bolt (with merge-fdata for multi-run aggregation) before BOLT.

Bug 4 — -reorder-blocks=cache+ is deprecated. Current upstream README and LLVM 19+ use ext-tsp. Fixed.

Bug 5 — BOLT flags were incomplete. -split-all-cold, -split-eh, -icf=1, and -use-gnu-stack are standard per BOLT's own OptimizingClang.md. All added.

Bug 6 — perf record -a -g --call-graph dwarf is wrong for BOLT/AutoFDO. -a profiles the whole system instead of the target binary. BOLT needs LBR branch records (-b flag), not DWARF call graphs. Corrected to perf record -e cycles:u -b -F <freq> -N.

Bug 7 — Multiple perf runs overwrote the same output file. Each iteration of the loop wrote to the same -o $PERF_DATA. Fixed with indexed filenames and merge-fdata aggregation.

Bug 8 — -fcoverage-mapping on the optimized rebuild. Coverage mapping is for code coverage reporting, not performance optimization, and bloats the binary. Removed from Step 4.

Bug 9 — perf stat -v usage. perf stat -v in this context doesn't mean what the comment claimed. It is kept but now correctly framed as a validation comparison tool, with || true guards since the measured binary may return nonzero.

Bug 10 — > >(tee ...) swallowed child exit codes under set -e. The run_logged helper now uses a subshell with wait to correctly propagate exit codes.

Bug 11 — src/*.c glob expansion was unguarded. Now uses find + mapfile -d '' with nullglob and an explicit empty-array check.

autofdo-pipeline

Fedora 43 · Clang instrumentation PGO → perf LBR sampling → LLVM BOLT post-link optimisation
Full stdout/stderr capture for every tool in the pipeline.
What this is

A self-contained, three-file repository that approximates and extends the
AutoFDO approach described in Android's Linux AutoFDO work.
File	Purpose
autofdo_full_capture.sh	Main pipeline: PGO → perf → BOLT
verify.sh	Parses logs, prints before/after perf delta table
Makefile	Generates a demo workload, wires make run / make verify / make lint

Prerequisites
bash

sudo dnf install clang llvm llvm-bolt perf

perf2bolt and merge-fdata ship inside the llvm-bolt package on Fedora 43.
LBR hardware requirement

BOLT's branch-record profiling (step 5) requires Last Branch Records (LBR):
Platform	Minimum
Intel	Haswell or later (most machines since ~2013)
AMD	Zen 3 with BRS, or Zen 4 with amd_lbr_v2
VM / container	LBR is usually not available — add -nl to the perf2bolt call in step 6

To check AMD LBR support:
bash

grep -E 'brs|amd_lbr_v2' /proc/cpuinfo

Quick start
bash

# 1. Clone / copy the three files into a directory
# 2. Point at your source tree, or let make generate the demo workload
make            # generates src/workload.c and validates it compiles
make run        # runs the full 8-step pipeline
make verify     # prints the delta report from the most recent run

Custom source tree
bash

SRC_DIR=/path/to/myproject/src \
APP_NAME=myapp \
WORKDIR=/tmp/autofdo \
LOG_DIR=/tmp/autofdo_logs \
bash autofdo_full_capture.sh

Tuning knobs

All variables have defaults; set them in the environment before running.
Variable	Default	Description
SRC_DIR	./src	Directory containing *.c source files
APP_NAME	my_app	Output binary base name
WORKDIR	./autofdo_workdir	PGO / BOLT artefacts
LOG_DIR	./autofdo_logs	Per-step stdout + stderr logs
PERF_RUNS	3	Number of LBR sampling runs
PERF_FREQ	2999	perf sampling frequency (Hz)
OPT_LEVEL	-O2	Clang optimisation flag
EXTRA_CFLAGS	(empty)	Appended to every clang invocation
Log layout

Every step writes two files:

autofdo_logs/
  step1_build_instr.stdout
  step1_build_instr.stderr      ← clang warnings/errors here
  step2_profiling_run.stdout    ← application stdout here
  step2_profiling_run.stderr
  step3_profdata_merge.stderr
  step4_build_pgo.stderr        ← clang PGO feedback messages
  step5_perf_run_1.stderr       ← perf verbose output
  step5_perf_run_N.stderr
  step6_perf2bolt.stderr        ← symbol resolution messages
  step6_merge_fdata.stderr
  step7_llvm_bolt.stderr        ← BOLT-INFO / BOLT-WARNING here
  step8_perf_stat_baseline.stderr
  step8_perf_stat_bolt.stderr

To watch BOLT do its work:
bash

grep 'BOLT-INFO' autofdo_logs/step7_llvm_bolt.stderr

Known limitations

DWARF v5 / BOLT: BOLT's -update-debug-sections is still incomplete for
DWARF v5 as of LLVM 19. The scripts force -gdwarf-4 to avoid this. Remove
that flag only if you do not need updated debug info in the BOLT binary.

Non-LBR environments: In virtual machines or containers where LBR is
unavailable, add -nl to the perf2bolt invocation in step 6 of
autofdo_full_capture.sh. Gains will be smaller (no taken-branch topology),
but the pipeline still produces a valid BOLT binary.

sudo perf record: Most Fedora 43 configurations require sudo for
perf record unless /proc/sys/kernel/perf_event_paranoid is set to ≤ 1.
To avoid sudo for user-space-only profiling:
bash

echo 1 | sudo tee /proc/sys/kernel/perf_event_paranoid
