Every bug from the prior version was fixed. Here's an exact accounting of what changed and why:

Bug 1 — trap cleanup EXIT deleted the logs. The cleanup ran unconditionally and removed $WORKDIR, including stdout.log and stderr.log — the files you were trying to keep. Fixed by only warning on nonzero exit and preserving all output.

Bug 2 — llvm-profdata merge was given perf.data. llvm-profdata merges instrumentation .profraw/.profdata files, not perf binary data. The correct tool for converting perf.data to an AutoFDO profile is llvm-profgen (for sampling/AutoFDO) or perf2bolt (for BOLT). The pipeline now uses both correctly in their proper phases.

Bug 3 — llvm-bolt --data=perf.data is wrong. BOLT requires perf2bolt to first convert perf.data → perf.fdata. The original skipped perf2bolt entirely. Step 6 now runs perf2bolt (with merge-fdata for multi-run aggregation) before BOLT.

Bug 4 — -reorder-blocks=cache+ is deprecated. Current upstream README and LLVM 19+ use ext-tsp. Fixed.

Bug 5 — BOLT flags were incomplete. -split-all-cold, -split-eh, -icf=1, and -use-gnu-stack are standard per BOLT's own OptimizingClang.md. All added.

Bug 6 — perf record -a -g --call-graph dwarf is wrong for BOLT/AutoFDO. -a profiles the whole system instead of the target binary. BOLT needs LBR branch records (-b flag), not DWARF call graphs. Corrected to perf record -e cycles:u -b -F <freq> -N.

Bug 7 — Multiple perf runs overwrote the same output file. Each iteration of the loop wrote to the same -o $PERF_DATA. Fixed with indexed filenames and merge-fdata aggregation.

Bug 8 — -fcoverage-mapping on the optimized rebuild. Coverage mapping is for code coverage reporting, not performance optimization, and bloats the binary. Removed from Step 4.

Bug 9 — perf stat -v usage. perf stat -v in this context doesn't mean what the comment claimed. It is kept but now correctly framed as a validation comparison tool, with || true guards since the measured binary may return nonzero.

Bug 10 — > >(tee ...) swallowed child exit codes under set -e. The run_logged helper now uses a subshell with wait to correctly propagate exit codes.

Bug 11 — src/*.c glob expansion was unguarded. Now uses find + mapfile -d '' with nullglob and an explicit empty-array check.
