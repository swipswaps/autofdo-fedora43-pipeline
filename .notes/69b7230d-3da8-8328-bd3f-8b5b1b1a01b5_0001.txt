🛠️ Compiler & Build‑Time Optimization Tools
1. GCC Profile‑Guided Optimization (PGO)

    What it is: Standard PGO supported by most compilers (including GCC on Fedora). This requires instrumenting a build, running representative workloads to collect profile data, and recompiling with that data to improve branch prediction, inlining, and layout decisions.

    Use on Fedora 43:

        GCC is the default system compiler (≥ 15.x).

        You can enable PGO with ‑fprofile‑generate/‑fprofile‑use.

        Typical workflow: build instrumented binary → run workloads → rebuild optimized.

    Strengths: Broad support, trusted, widely used beyond just kernel builds.

2. LLVM/Clang PGO and Sampling‑Based Optimization

    What it is: LLVM supports both instrumentation PGO and sampling‑based feedback (AutoFDO‑style), converting perf sampling profiles to guide optimization.

Fedora 43 specifics:

    Fedora is updating its LLVM toolchain (Clang 21) and builds LLVM itself with PGO enabled to improve compilation performance.

    Fedora packages include llvm, clang, and related tools.

How to use (developer side):

    Collect perf profiles with a workload (e.g., perf record …).

    Use tools like llvm‑profgen or Google’s create_llvm_prof from the autofdo project to turn perf data into PGO profiles for use with Clang.

    Pros: More automated than classical PGO; can use real execution profiles.

Packages you’ll likely install via dnf:

sudo dnf install llvm clang llvm-bolt llvm-devel

(llvm-bolt is next — see below)
🧠 Post‑Link / Binary Optimizers
3. LLVM BOLT (Binary Optimization and Layout Tool)

    What it is: A post‑link, profile‑guided binary optimizer. It takes compiled binaries and reorganizes functions and basic blocks based on profiles, improving cache behavior and branch locality without recompiling from source.

Fedora availability:

    Fedora’s repositories include llvm‑bolt (and docs).

    How it fits: Complements PGO/AutoFDO by optimizing layout after compilation.

Use case (simplified):

perf record -d -d -d ./myapp
llvm-bolt ./myapp -o myapp.bolt --perf-data=perf.data

🧪 Additional Profiling & Tuning Tools

These aren’t compiler plug‑ins like AutoFDO, but they are essential in profiling and validating optimization decisions:
4. perf (Linux Performance Counters)

    Role: Collects hardware sampling data (branches, cache interactions) needed for sampling‑based PGO and BOLT profiles.

    Use: Base tool for profile gathering. You’ll use it to produce .data that drives feedback‑based optimization.

    sudo dnf install perf

5. Phoronix Test Suite

    Role: Benchmarking framework that helps you measure the performance impact of your optimization experiments (from compiler flags to binary reordering).

    Useful to validate before/after performance.

There has been discussion about ramping up PGO and BOLT usage across Fedora by default, but it’s still largely developer‑driven and not systemic across all packages.

#!/usr/bin/env bash
# autofdo_full_capture.sh
# Fedora 43 – Full visibility AutoFDO-style profiling
# Captures all runtime events, system messages, errors, and hidden outputs
# Combines GCC/Clang PGO + perf + LLVM BOLT with full stdout/stderr logging
# Strict linting, safe flags, and PRF-style verbose comments

set -euo pipefail
IFS=$'\n\t'  # Safe IFS for loops and filenames

# -------------------------------
# CONFIGURATION
# -------------------------------

# Binary to profile – adjust as needed
TARGET_BINARY="./my_app"

# Temp directories for profiles and logs
WORKDIR="./autofdo_workdir"
mkdir -p "$WORKDIR"

# Instrumentation output
PGO_PROFILE="$WORKDIR/pgo.prof"
PERF_DATA="$WORKDIR/perf.data"
BOLT_BINARY="$WORKDIR/my_app.bolt"

# Log files
STDOUT_LOG="$WORKDIR/stdout.log"
STDERR_LOG="$WORKDIR/stderr.log"

# Number of runs for perf sampling to generate representative profile
PERF_RUNS=3

# -------------------------------
# CLEANUP FUNCTION
# -------------------------------
cleanup() {
    echo "Cleaning up temporary files..." >&2
    rm -rf "$WORKDIR"
}
trap cleanup EXIT

# -------------------------------
# STEP 1: BUILD INSTRUMENTED BINARY (PGO)
# -------------------------------
# Compile target with PGO instrumentation
# GCC/Clang: generate profile data
# Output goes to stdout/stderr automatically
# Safe: do not overwrite existing build unless WORKDIR is fresh

echo "[INFO] Step 1: Building instrumented binary for PGO..." >&2
if [[ -f "$TARGET_BINARY" ]]; then
    echo "[INFO] Existing binary detected, cleaning..." >&2
    rm -f "$TARGET_BINARY"
fi

# Example: Clang PGO instrumented build
# Adjust CFLAGS/LDFLAGS as needed for your project
clang -O2 -fprofile-instr-generate -fcoverage-mapping -o "$TARGET_BINARY" src/*.c \
    > >(tee -a "$STDOUT_LOG") \
    2> >(tee -a "$STDERR_LOG" >&2)

# -------------------------------
# STEP 2: COLLECT PERF DATA
# -------------------------------
# Run binary under perf with full verbose capture
# Collect hardware events, branches, CPU cycles, cache misses
# Capture stdout/stderr in logs and terminal simultaneously

echo "[INFO] Step 2: Collecting perf profile ($PERF_RUNS runs)..." >&2
for i in $(seq 1 "$PERF_RUNS"); do
    echo "[INFO] Perf run $i/$PERF_RUNS" >&2
    sudo perf record -v -a -g --call-graph dwarf -o "$PERF_DATA" "$TARGET_BINARY" \
        > >(tee -a "$STDOUT_LOG") \
        2> >(tee -a "$STDERR_LOG" >&2)
done

# -------------------------------
# STEP 3: GENERATE LLVM PGO PROFILE (AutoFDO)
# -------------------------------
# Convert perf.data to AutoFDO-readable format
# Requires LLVM/Clang + autofdo tools installed

echo "[INFO] Step 3: Generating LLVM AutoFDO profile..." >&2
llvm-profdata merge -output="$PGO_PROFILE" "$PERF_DATA" \
    > >(tee -a "$STDOUT_LOG") \
    2> >(tee -a "$STDERR_LOG" >&2)

# -------------------------------
# STEP 4: REBUILD BINARY WITH PGO FEEDBACK
# -------------------------------
# Use the generated profile to guide optimizations
# This is the AutoFDO equivalent of Android's approach

echo "[INFO] Step 4: Rebuilding binary with PGO feedback..." >&2
clang -O2 -fprofile-instr-use="$PGO_PROFILE" -fcoverage-mapping -o "$TARGET_BINARY" src/*.c \
    > >(tee -a "$STDOUT_LOG") \
    2> >(tee -a "$STDERR_LOG" >&2)

# -------------------------------
# STEP 5: POST-LINK OPTIMIZATION WITH LLVM BOLT
# -------------------------------
# Reorganize functions and basic blocks for better cache locality
# Output is BOLT-optimized binary
# Verbose mode ensures all messages are visible

echo "[INFO] Step 5: Running LLVM BOLT post-link optimization..." >&2
llvm-bolt "$TARGET_BINARY" \
    -o "$BOLT_BINARY" \
    --data="$PERF_DATA" \
    -reorder-blocks=cache+ \
    -reorder-functions=hfsort \
    -split-functions \
    -dyno-stats \
    -v \
    > >(tee -a "$STDOUT_LOG") \
    2> >(tee -a "$STDERR_LOG" >&2)

# -------------------------------
# STEP 6: VALIDATE BINARY AND CAPTURE FINAL MESSAGES
# -------------------------------
# Run final optimized binary once to verify execution
# Capture all stdout, stderr, and hidden system messages

echo "[INFO] Step 6: Running final BOLT-optimized binary for validation..." >&2
sudo perf stat -v "$BOLT_BINARY" \
    > >(tee -a "$STDOUT_LOG") \
    2> >(tee -a "$STDERR_LOG" >&2)

echo "[INFO] AutoFDO-style profiling and full capture complete." >&2
echo "[INFO] Logs available in $WORKDIR/stdout.log and $WORKDIR/stderr.log" >&2
